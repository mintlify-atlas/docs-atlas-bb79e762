---
title: "Routes API — plan and manage delivery routes"
description: "Create and manage delivery routes with OpenRouteService integration, ordered stops, and automatic geofence generation from computed geometry."
---

The Routes API lets you manually create delivery routes or auto-plan them via OpenRouteService (ORS). Each route belongs to an order (`orden_id`) and can carry an ordered list of stops (`paradas`). When auto-planning, the API calls ORS to compute distance and duration, then optionally creates a buffer geofence around the computed geometry in MongoDB.

<Info>All endpoints require a valid Bearer token. The required permission is noted on each operation.</Info>

---

## Create route

`POST /rutas`

Manually create a route with a set of stops.

**Permission:** `rutas:write`

<ParamField body="orden_id" type="integer" required>
  ID of the order this route belongs to. Must reference an existing order.
</ParamField>

<ParamField body="distancia_km" type="number">
  Total route distance in kilometres.
</ParamField>

<ParamField body="tiempo_estimado" type="string">
  Estimated travel time as an ISO 8601 duration string (e.g. `"PT45M"`).
</ParamField>

<ParamField body="paradas" type="object[]">
  Ordered list of stops to include on creation.

  <Expandable title="parada fields">
    <ParamField body="direccion" type="string" required>
      Street address of the stop.
    </ParamField>
    <ParamField body="secuencia" type="integer" required>
      Position of this stop in the route sequence (1-based).
    </ParamField>
    <ParamField body="orden_id" type="integer">
      Order ID associated with this particular stop (can differ from the route's `orden_id`).
    </ParamField>
    <ParamField body="distrito" type="string">
      District or neighbourhood name.
    </ParamField>
    <ParamField body="estado" type="string" default="Pendiente">
      Stop status. One of `Pendiente`, `Visitada`, `Omitida`.
    </ParamField>
  </Expandable>
</ParamField>

<RequestExample>
```bash cURL
curl -X POST "https://api.rappi2.com/rutas" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "orden_id": 42,
    "distancia_km": 12.5,
    "tiempo_estimado": "PT30M",
    "paradas": [
      {"direccion": "Av. Arequipa 1234", "distrito": "Miraflores", "secuencia": 1}
    ]
  }'
```
</RequestExample>

<ResponseExample>
```json 201
{
  "id": 7,
  "orden_id": 42,
  "distancia_km": "12.500",
  "tiempo_estimado": "PT30M",
  "paradas": [
    {
      "id": 1,
      "ruta_id": 7,
      "orden_id": 42,
      "direccion": "Av. Arequipa 1234",
      "distrito": "Miraflores",
      "secuencia": 1,
      "estado": "Pendiente",
      "fecha_paso": null
    }
  ]
}
```
</ResponseExample>

## Auto-plan route via ORS

`POST /rutas/planificar`

Calls OpenRouteService to compute the optimal route between an origin and destination, stores the computed distance and travel time, and optionally creates a buffer geofence in MongoDB around the returned geometry.

**Permission:** `rutas:write`

<Warning>
  Returns `502` if the ORS service is unreachable or returns an error.
</Warning>

<ParamField body="orden_id" type="integer" required>
  Order to associate with the planned route.
</ParamField>

<ParamField body="origen_lon" type="number" required>
  Origin longitude (`-180` to `180`).
</ParamField>

<ParamField body="origen_lat" type="number" required>
  Origin latitude (`-90` to `90`).
</ParamField>

<ParamField body="destino_lon" type="number" required>
  Destination longitude (`-180` to `180`).
</ParamField>

<ParamField body="destino_lat" type="number" required>
  Destination latitude (`-90` to `90`).
</ParamField>

<ParamField body="generar_geocerca" type="boolean" default="true">
  When `true`, a `ruta_buffer` geofence is created in MongoDB around the computed route geometry.
</ParamField>

<ParamField body="tolerancia_metros" type="integer" default="50">
  Buffer tolerance in metres for the geofence polygon (`1`–`5000`).
</ParamField>

<RequestExample>
```bash cURL
curl -X POST "https://api.rappi2.com/rutas/planificar" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "orden_id": 42,
    "origen_lon": -77.0282,
    "origen_lat": -12.0432,
    "destino_lon": -77.0531,
    "destino_lat": -12.1219,
    "generar_geocerca": true,
    "tolerancia_metros": 100
  }'
```
</RequestExample>

<ResponseExample>
```json 201
{
  "id": 8,
  "orden_id": 42,
  "distancia_km": "9.830",
  "tiempo_estimado": "PT22M30S",
  "paradas": []
}
```
</ResponseExample>

## Delete route

`DELETE /rutas/{ruta_id}`

Deletes the route and all associated geofences stored in MongoDB.

**Permission:** `rutas:delete`

<Warning>Deleting a route also removes all geofences linked to it from the `geocercas` collection in MongoDB.</Warning>

<ParamField path="ruta_id" type="integer" required>
  Route ID.
</ParamField>

<RequestExample>
```bash cURL
curl -X DELETE "https://api.rappi2.com/rutas/7" \
  -H "Authorization: Bearer <token>"
```
</RequestExample>

<ResponseExample>
```json 204
```
</ResponseExample>

## Response schema

<ResponseField name="id" type="integer" required>
  Route ID (auto-incremented).
</ResponseField>

<ResponseField name="orden_id" type="integer" required>
  Order this route belongs to.
</ResponseField>

<ResponseField name="distancia_km" type="string">
  Distance in kilometres, returned as a decimal string.
</ResponseField>

<ResponseField name="tiempo_estimado" type="string">
  Estimated travel time in ISO 8601 duration format (e.g. `"PT30M"`).
</ResponseField>

<ResponseField name="paradas" type="object[]">
  Ordered list of stops on this route.

  <Expandable title="parada fields">
    <ResponseField name="id" type="integer" required>Stop ID.</ResponseField>
    <ResponseField name="ruta_id" type="integer" required>Parent route ID.</ResponseField>
    <ResponseField name="orden_id" type="integer">Associated order ID.</ResponseField>
    <ResponseField name="direccion" type="string" required>Street address.</ResponseField>
    <ResponseField name="distrito" type="string">District or neighbourhood.</ResponseField>
    <ResponseField name="secuencia" type="integer" required>Sequence position (1-based).</ResponseField>
    <ResponseField name="estado" type="string" required>
      One of `Pendiente`, `Visitada`, `Omitida`.
    </ResponseField>
    <ResponseField name="fecha_paso" type="string">
      ISO 8601 datetime when the stop was visited. `null` if not yet visited.
    </ResponseField>
  </Expandable>
</ResponseField>
