---
title: "Audit API — query HTTP audit logs"
description: "Query the MongoDB audit log of all HTTP requests made to the API, with filters by user and method, and an aggregated summary endpoint."
---

Every HTTP request processed by the API is recorded in the `auditoria` MongoDB collection by a middleware layer before the response is returned. Each document captures the request method, route, caller identity, response status code, client IP, and a SHA hash of the request payload. Records are automatically purged after 90 days via a MongoDB TTL index.

<Info>All endpoints require `Authorization: Bearer <token>` and the `auditoria:read` permission.</Info>

---

## Audit log document fields

Each audit document stored in MongoDB contains the following fields:

| Field | Type | Description |
|---|---|---|
| `_id` | ObjectId | MongoDB document ID (returned as string `id`). |
| `usuario_id` | integer | Authenticated user ID, or `null` for unauthenticated requests. |
| `ruta` | string | Request path (e.g. `/rutas/7`). |
| `metodo` | string | HTTP method: `GET`, `POST`, `PATCH`, `DELETE`, etc. |
| `ip` | string | Client IP address, or `null` if not resolvable. |
| `status_code` | integer | HTTP response status code. |
| `payload_hash` | string | SHA hash of the request body, or `null` for requests without a body. |
| `timestamp` | datetime | UTC time the request was processed. Auto-expires after 90 days. |

## List audit entries

`GET /auditoria`

Returns audit log entries in descending timestamp order with optional filters.

**Permission:** `auditoria:read`

<ParamField query="usuario_id" type="integer">
  Filter to entries generated by a specific user.
</ParamField>

<ParamField query="metodo" type="string">
  Filter by HTTP method (e.g. `"POST"`, `"DELETE"`).
</ParamField>

<ParamField query="skip" type="integer" default="0">
  Pagination offset.
</ParamField>

<ParamField query="limit" type="integer" default="100">
  Maximum entries to return (max `500`).
</ParamField>

<RequestExample>
```bash cURL
curl -X GET "https://api.rappi2.com/auditoria?metodo=DELETE&limit=20" \
  -H "Authorization: Bearer <token>"
```
</RequestExample>

<ResponseExample>
```json 200
[
  {
    "id": "664f4d5e6f7a8b9c0d1e2f30",
    "usuario_id": 3,
    "ruta": "/rutas/7",
    "metodo": "DELETE",
    "ip": "190.41.55.12",
    "status_code": 204,
    "payload_hash": null,
    "timestamp": "2026-05-22T14:20:00Z"
  },
  {
    "id": "664f4d5e6f7a8b9c0d1e2f31",
    "usuario_id": 3,
    "ruta": "/incidencias",
    "metodo": "POST",
    "ip": "190.41.55.12",
    "status_code": 201,
    "payload_hash": "a3f2c1d4e5b6a7c8d9e0f1a2b3c4d5e6",
    "timestamp": "2026-05-22T13:45:10Z"
  }
]
```
</ResponseExample>

## Audit summary

`GET /auditoria/resumen`

Aggregated statistics over a time window: request count by status code, by method, top 10 most-requested routes, top 10 most-active users, and error breakdown (4xx/5xx).

**Permission:** `auditoria:read`

<ParamField query="horas" type="integer" default="24">
  Analysis window in hours (`1`–`720`, i.e. up to 30 days).
</ParamField>

<RequestExample>
```bash cURL
curl -X GET "https://api.rappi2.com/auditoria/resumen?horas=48" \
  -H "Authorization: Bearer <token>"
```
</RequestExample>

<ResponseExample>
```json 200

    "201": 810,
    "204": 390,

    "422": 40,

    {"ruta": "/tracking/ping", "requests": 2800},
```

<ResponseField name="ventana_horas" type="integer" required>
  The analysis window that was applied.
</ResponseField>

<ResponseField name="desde" type="string" required>
  ISO 8601 start of the window.
</ResponseField>

<ResponseField name="total_requests" type="integer" required>
  Total number of requests in the window.
</ResponseField>

<ResponseField name="by_status" type="object" required>
  Request count keyed by HTTP status code (as strings).
</ResponseField>

<ResponseField name="by_metodo" type="object" required>
  Request count keyed by HTTP method.
</ResponseField>

<ResponseField name="top_rutas" type="object[]" required>
  Top 10 most-requested routes.
  <Expandable title="fields">
    <ResponseField name="ruta" type="string">Request path.</ResponseField>
    <ResponseField name="requests" type="integer">Number of requests.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="top_usuarios" type="object[]" required>
  Top 10 most-active authenticated users.
  <Expandable title="fields">
    <ResponseField name="usuario_id" type="integer">User ID.</ResponseField>
    <ResponseField name="requests" type="integer">Number of requests.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="errores_4xx_5xx" type="object" required>
  Error request count keyed by status code (only codes >= 400).
</ResponseField>

## Audit entry response schema

<ResponseField name="id" type="string" required>
  MongoDB ObjectId as a hex string.
</ResponseField>

<ResponseField name="usuario_id" type="integer">
  Authenticated user ID. `null` for unauthenticated requests.
</ResponseField>

<ResponseField name="ruta" type="string" required>
  Request path.
</ResponseField>

<ResponseField name="metodo" type="string" required>
  HTTP method.
</ResponseField>

<ResponseField name="ip" type="string">
  Client IP address.
</ResponseField>

<ResponseField name="status_code" type="integer" required>
  HTTP response status code.
</ResponseField>

<ResponseField name="payload_hash" type="string">
  SHA hash of the request body. `null` for GET/DELETE requests or empty bodies.
</ResponseField>

<ResponseField name="timestamp" type="string" required>
  ISO 8601 UTC timestamp of the request. Documents are automatically removed after 90 days.
</ResponseField>
