---
title: "Reports API — KPIs, SLA, and analytics endpoints"
description: "Read-only analytics endpoints: dashboard KPIs, sales series, SLA compliance, driver efficiency, incident summaries, and real-time operational status across Postgres and MongoDB."
---

The Reports API provides pre-computed analytics by querying PostgreSQL (orders, assignments, conductors, payments) and MongoDB (tracking, geofences, evidences, notifications). All endpoints are read-only and require the `reportes:read` permission.

<Info>All endpoints require `Authorization: Bearer <token>` and the `reportes:read` permission.</Info>

---

## Dashboard

`GET /reportes/dashboard`

High-level operational KPIs for the past 24 hours and overall entity counts.

<RequestExample>
```bash cURL
curl -X GET "https://api.rappi2.com/reportes/dashboard" \
  -H "Authorization: Bearer <token>"
```
</RequestExample>

<ResponseExample>
```json 200
{
  "totales": {
    "clientes": 142,
    "usuarios": 18,
    "conductores": 34,
    "vehiculos": 29,
    "ordenes": 1820,
    "asignaciones": 1750
  },
  "ordenes_por_estado": {
    "Pendiente": 42,
    "En Proceso": 15,
    "En Tránsito": 8,
    "Entregado": 1700,
    "Cancelado": 55
  },
  "conductores_por_disponibilidad": {
    "Disponible": 20,
    "Ocupado": 12,
    "Inactivo": 2
  },
  "vehiculos_por_estado": {
    "Operativo": 25,
    "Mantenimiento": 3,
    "Inactivo": 1
  },
  "recaudacion_ultimas_24h": 4820.50,
  "incidencias_severidad_alta": 3
```

<ResponseField name="totales" type="object" required>
  Count of active entities per type.
  <Expandable title="fields">
    <ResponseField name="clientes" type="integer">Active clients.</ResponseField>

  Count of incidents with `severidad >= 3`.

## Sales report

`GET /reportes/ventas`

Revenue aggregated by day or month. Defaults to the last 30 days when `desde`/`hasta` are omitted.

<ParamField query="desde" type="string">
  ISO 8601 datetime — start of the reporting window. Defaults to 30 days before `hasta`.
</ParamField>

<ParamField query="hasta" type="string">
  ISO 8601 datetime — end of the reporting window. Defaults to now.
</ParamField>

<ParamField query="granularidad" type="string" default="dia">
  Aggregation bucket: `dia` (daily) or `mes` (monthly).
</ParamField>

<RequestExample>
```bash cURL
curl -X GET "https://api.rappi2.com/reportes/ventas?granularidad=mes&desde=2026-01-01T00:00:00Z" \
  -H "Authorization: Bearer <token>"
```
</RequestExample>

<ResponseExample>
```json 200
{
  "desde": "2026-01-01T00:00:00Z",
  "hasta": "2026-05-22T14:00:00Z",
  "granularidad": "mes",
  "series": [
    {"periodo": "2026-01-01T00:00:00Z", "pagos": 320, "monto": 48200.00},
    {"periodo": "2026-02-01T00:00:00Z", "pagos": 290, "monto": 43500.00}
  ],
  "total_recaudado": 91700.00,
  "total_facturado": 95000.00
}
```
</ResponseExample>

## Driver efficiency

`GET /reportes/conductores/eficiencia`

Per-driver efficiency metrics: deliveries per hour and incident rate, for active drivers only.

<ParamField query="desde" type="string">
  ISO 8601 datetime — start of the analysis window.
</ParamField>

<ParamField query="hasta" type="string">
  ISO 8601 datetime — end of the analysis window.
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Maximum rows to return (`1`–`200`). Ordered by `entregas_por_hora` descending.
</ParamField>

<RequestExample>
```bash cURL
curl -X GET "https://api.rappi2.com/reportes/conductores/eficiencia?desde=2026-05-01T00:00:00Z&limit=10" \
  -H "Authorization: Bearer <token>"
```
</RequestExample>

<ResponseExample>
```json 200
[
  {
    "conductor_id": 5,
    "nombre": "Carlos Quispe",
    "entregas_finalizadas": 82,
    "horas_activas": 134.5,
    "entregas_por_hora": 0.61,
    "incidencias": 3,
    "incidencias_severas": 1,
    "tasa_incidencias": 0.037
  }
]
```
</ResponseExample>

<ResponseField name="conductor_id" type="integer" required>Driver ID.</ResponseField>
<ResponseField name="nombre" type="string" required>Driver name.</ResponseField>
<ResponseField name="entregas_finalizadas" type="integer" required>Number of completed deliveries in the window.</ResponseField>
<ResponseField name="horas_activas" type="number" required>Total active hours (sum of assignment durations).</ResponseField>
<ResponseField name="entregas_por_hora" type="number">Deliveries per active hour. `null` when `horas_activas` is 0.</ResponseField>
<ResponseField name="incidencias" type="integer" required>Total incidents across all assignments.</ResponseField>
<ResponseField name="incidencias_severas" type="integer" required>Incidents with `severidad >= 3`.</ResponseField>
<ResponseField name="tasa_incidencias" type="number">Incidents per delivery. `null` when `entregas_finalizadas` is 0.</ResponseField>

## SLA compliance

`GET /reportes/sla-entregas`

Percentage of completed assignments delivered within a configurable SLA threshold, plus p50/p95 duration percentiles.

<ParamField query="desde" type="string">
  ISO 8601 datetime — start of the window.
</ParamField>

<ParamField query="hasta" type="string">
  ISO 8601 datetime — end of the window.
</ParamField>

<ParamField query="sla_minutos" type="integer" default="60">
  SLA target in minutes (`1`–`1440`). Assignments completed within this threshold count as on-time.
</ParamField>

<RequestExample>
```bash cURL
curl -X GET "https://api.rappi2.com/reportes/sla-entregas?sla_minutos=45" \
  -H "Authorization: Bearer <token>"
```
</RequestExample>

<ResponseExample>
```json 200
{
  "desde": null,
  "hasta": null,
  "sla_minutos": 45,
  "total_entregas": 1750,
  "on_time": 1435,
  "off_time": 315,
  "on_time_pct": 82.0,
  "p50_minutos": 31.5,
  "p95_minutos": 67.2
}
```
</ResponseExample>

<ResponseField name="on_time_pct" type="number">
  Percentage of deliveries completed within `sla_minutos`. `null` when there are no deliveries.
</ResponseField>

<ResponseField name="p50_minutos" type="number">
  Median delivery duration in minutes. `null` when there are no deliveries.
</ResponseField>

<ResponseField name="p95_minutos" type="number">
  95th-percentile delivery duration in minutes. `null` when there are no deliveries.
</ResponseField>

## Operational report

`GET /reportes/operativo`

Real-time cross-source snapshot combining PostgreSQL (active assignments, driver availability) and MongoDB (online drivers, GPS coverage, geofences, recent incidents).

<ParamField query="ventana_minutos" type="integer" default="5">
  Time window in minutes for considering a driver "online" based on recent pings (`1`–`60`).
</ParamField>

<RequestExample>
```bash cURL
curl -X GET "https://api.rappi2.com/reportes/operativo?ventana_minutos=10" \
  -H "Authorization: Bearer <token>"
```
</RequestExample>

<ResponseExample>
```json 200
{
  "ventana_minutos": 10,
  "asignaciones_en_curso": 12,
  "conductores_ocupados_postgres": 12,
  "conductores_online_mongo": 10,
  "asignaciones_activas_sin_tracking": 2,
  "asignaciones_con_ping_hoy": 34,
  "geocercas_activas": 18,
  "incidencias_ultimas_24h": 5
}
```
</ResponseExample>

<ResponseField name="conductores_online_mongo" type="integer" required>
  Distinct driver IDs with a ping within the `ventana_minutos` window.
</ResponseField>

<ResponseField name="asignaciones_activas_sin_tracking" type="integer" required>
  Active assignments (`EnCurso`) with no GPS ping in the current window.
</ResponseField>

<ResponseField name="asignaciones_con_ping_hoy" type="integer" required>
  Distinct assignments that have received at least one ping today (since midnight UTC).
</ResponseField>
